# Game Design Document

## Overview

| | |
|---|---|
| Engine | Godot 4.4, GDScript only |
| Renderer | GL Compatibility (`renderer/rendering_method="gl_compatibility"`) |
| Platform | Desktop (PC); no mobile/controller input configured |
| Genre | Resource-management drive / score-attack |
| Session length | One run: a few minutes, ends when the tank runs dry |
| Viewport | 960×360 internal, window override 1440×540, `stretch/mode="canvas_items"`, aspect `keep` |

See `00_VISION.md` for the player fantasy and design pillars.

## Controls

The player does not drive. The full input surface is two actions:

| Action | Key(s) | Effect |
|---|---|---|
| `refuel` | E | Refuel while inside a gas station's trigger zone |
| `restart` | R, Enter | Restart the run from the Game Over screen |

(`project.godot` `[input]`; no movement/lane input exists in the current build.)

## Win / lose condition

There is no win state — the game is a score-attack on distance driven. A run
ends exactly one way: **OUT_OF_FUEL** (`GameManager.RunEndReason`), when the
tank empties and the car rolls to a stop. Money hitting zero does not end a
run by itself; it just means the player can no longer afford to refuel,
which makes running out of fuel later inevitable. See `03_CORE_MECHANICS.md`.

## Core loop

Summarized in `02_GAME_LOOP.md`: drive (automatic) → fuel drains with
distance → reach a gas station → decide whether to refuel → repeat until the
tank is empty.

## Systems

Detailed in `03_CORE_MECHANICS.md`: drivetrain/movement, fuel, money, gas
stations, world-scale conversion, telemetry, end/restart.

## UI

- **HUD** (`ui/hud.tscn`, `ui/hud.gd`): speed (km/h), distance (m/km), fuel
  (%), money ($). Pure presentation — Main pushes formatted values in via
  signals; the HUD holds no game state.
- **Game Over screen** (`ui/game_over.tscn`, `ui/game_over.gd`): shown when a
  run ends. Displays "OUT OF FUEL" and the distance driven
  (`UiFormat.distance_text`), and listens for the `restart` action.

## Art style

2D pixel art, side-view, parallax-scrolled (sky / mountains / forest /
foreground layers plus a scrolling road). All current art is placeholder,
procedurally generated by `tools/generate_placeholder_art.py` — nothing is
hand-authored yet. See `11_TECHNICAL_ARCHITECTURE.md`.

## Scope statement

**Shipped:** Day 1 (driving feel, endless road streaming, camera) and Day 2
(fuel, money, gas stations, HUD, Game Over/restart), followed by a same-repo
pivot that removed the original lane-switching/traffic/crash layer entirely
in favor of the pure fuel-survival loop described here.

**Not built:** anything beyond the current loop — no income, no roadside
events, no NPCs/cities/regions/items/story, no save system. See
`12_ROADMAP.md` for what's actually planned versus explicitly out of scope.
